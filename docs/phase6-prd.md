## 1. 文档概述

### 1.1 文档目的

本文档定义项目资源计划预警系统 Phase 6 的详细产品需求，涵盖：
- **P1-002**: 分包队伍计划管理（24h）
- **P1-003**: 劳动力需求管理（24h）

### 1.2 业务背景

在大型工程项目中，分包队伍和劳动力是核心资源要素。当前系统已实现物资资源计划管理，但缺乏：
1. **分包队伍计划管理** - 无法有效跟踪分包队伍的进场、工作进展和退场
2. **劳动力需求管理** - 无法按工种和劳务类别进行精细化人力规划

Phase 6 将填补这两个关键模块，实现资源计划的完整覆盖。

### 1.3 与其他阶段的关系

| 阶段 | 内容 | 与 Phase 6 的关系 |
|------|------|-------------------|
| Phase 1-5 | 物资资源计划、预警、审批 | Phase 6 复用相同的预警机制和审批流程 |
| Phase 6 | 分包+劳动力管理 | 扩展资源计划类型，复用现有基础设施 |
| Phase 7+ | 看板、自定义报表 | 基于 Phase 6 数据生成统计看板 |

## 3. 功能需求

### 3.1 分包队伍计划管理（P1-002）

#### 3.1.1 页面结构

```
📊 项目资源计划预警系统
└── 📁 资源计划管理
    └── 分包队伍需求计划          ← 新增菜单项
        ├── 分包队伍计划列表
        ├── 新增/编辑分包计划
        └── 分包计划详情
```

#### 3.1.2 分包队伍计划列表页

**页面元素**：

| 区域 | 元素 | 说明 |
|------|------|------|
| 筛选区 | 项目选择下拉框 | 项目级用户默认当前项目，公司级可选下属项目 |
| 筛选区 | WBS编码输入框 | 支持模糊搜索 |
| 筛选区 | 分包名称输入框 | 支持模糊搜索 |
| 筛选区 | 供应商下拉框 | 从供应商字典选择 |
| 筛选区 | 计划时间范围 | 起始日期-结束日期 |
| 筛选区 | 状态筛选 | 全部/未开始/进行中/已完成 |
| 操作区 | [新增]按钮 | 打开新增页面 |
| 操作区 | [导入]按钮 | 导入Excel模板 |
| 操作区 | [导出]按钮 | 导出当前列表 |
| 操作区 | [批量删除]按钮 | 删除选中项（草稿状态） |
| 表格区 | 复选框 | 支持多选 |
| 表格区 | 列：项目编号、项目名称、WBS编码、WBS名称、分包项目名称、工作内容、供应商、计划开始日期、计划结束日期、实际开始日期、实际结束日期、状态、操作 |
| 表格区 | 操作列：[查看][编辑][删除][发起变更] | 根据状态显示可用操作 |
| 分页区 | 分页控件 | 默认每页20条 |

**状态规则**：

| 状态 | 判定条件 | 可用操作 |
|------|----------|----------|
| 草稿 | 未提交 | [编辑][删除][提交] |
| 进行中 | 实际开始日期≠空 且 实际结束日期=空 | [登记进展][发起变更] |
| 已完成 | 实际结束日期≠空 | [查看] |
| 变更中 | 存在未完成审批流程 | [查看审批进度] |

#### 3.1.3 新增/编辑分包计划页

**表单字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 项目 | 下拉选择 | 是 | 从用户有权限的项目中选择 |
| WBS编码 | 级联选择 | 是 | 级联：类型→WBS编号→WBS名称自动带出 |
| 分包项目名称 | 文本输入 | 是 | 最多100字符 |
| 工作内容 | 文本域 | 否 | 详细描述分包工作内容 |
| 供应商 | 下拉选择 | 是 | 从供应商字典选择 |
| 计划开始日期 | 日期选择 | 是 | - |
| 计划结束日期 | 日期选择 | 是 | 必须晚于计划开始日期 |
| 备注 | 文本域 | 否 | - |

**校验规则**：
1. 计划结束日期 > 计划开始日期
2. 同一项目下，同一WBS编码+分包项目名称不能重复（排除自身）
3. 供应商必须从有效供应商字典中选择

**按钮**：
- [保存草稿] - 保存为草稿状态
- [提交] - 提交生效（需填写完整必填项）
- [取消] - 返回列表

#### 3.1.4 分包计划详情/进展登记页

**页面布局**：

```
┌──────────────────────────────────────────────────────┐
│ 分包队伍计划详情                     [发起变更][返回] │
├──────────────────────────────────────────────────────┤
│ 基本信息                                            │
│ 项目：XX项目     WBS：WBS-001  分包项目：桩基分包    │
│ 供应商：XX公司   计划周期：2026-05-01 ~ 2026-06-30  │
│ 状态：进行中                                      │
├──────────────────────────────────────────────────────┤
│ 进展登记                                  [登记进展] │
│ 实际开始日期：[2026-05-02]                          │
│ 实际结束日期：[         ]    ← 未完成时不填          │
│ 备注：[                             ]               │
├──────────────────────────────────────────────────────┤
│ 变更记录                                            │
│ 序号  变更类型   变更内容        申请人  申请时间  状态│
│  1   修改时间   开始时间延期3天 张三   2026-05-10  已通过│
└──────────────────────────────────────────────────────┘
```

**进展登记逻辑**：
- 实际开始日期：首次登记时填写，不可修改（除非通过变更流程）
- 实际结束日期：分包工作完成时填写，填写后状态变为"已完成"

#### 3.1.5 分包计划变更流程

复用 Phase 1-5 的审批流程基础设施（Flowable），变更类型包括：

| 变更类型 | 可修改字段 | 需要审批 |
|----------|------------|----------|
| 修改时间 | 计划开始日期、计划结束日期 | 是 |
| 修改供应商 | 供应商 | 是 |
| 修改工作内容 | 工作内容 | 否（直接生效） |
| 终止分包 | 状态改为"已终止" | 是 |

**变更申请表单**：
- 变更类型（单选）
- 变更原因（必填，文本域）
- 原值/新值对比展示

### 4.2 数据库表设计

#### 4.2.1 resource_plan_subcontract（分包计划表）

```sql
CREATE TABLE resource_plan_subcontract (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键',
    project_id BIGINT NOT NULL COMMENT '项目ID',
    wbs_code VARCHAR(100) NOT NULL COMMENT 'WBS编码',
    wbs_name VARCHAR(200) COMMENT 'WBS名称',
    subcontract_name VARCHAR(100) NOT NULL COMMENT '分包项目名称',
    work_content TEXT COMMENT '工作内容',
    supplier_code VARCHAR(50) NOT NULL COMMENT '供应商编码',
    supplier_name VARCHAR(200) COMMENT '供应商名称',
    plan_start_date DATE NOT NULL COMMENT '计划开始日期',
    plan_end_date DATE NOT NULL COMMENT '计划结束日期',
    actual_start_date DATE COMMENT '实际开始日期',
    actual_end_date DATE COMMENT '实际结束日期',
    status VARCHAR(50) DEFAULT 'draft' COMMENT '状态',
    remark VARCHAR(500) COMMENT '备注',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    created_by BIGINT COMMENT '创建人',
    updated_by BIGINT COMMENT '更新人',
    UNIQUE KEY uk_project_wbs_subcontract (project_id, wbs_code, subcontract_name),
    INDEX idx_project (project_id),
    INDEX idx_status (status)
) COMMENT='分包队伍计划表';
```

## 5. 预警机制设计

### 5.1 分包队伍计划预警

#### 5.1.1 预警规则

| 预警类型 | 触发条件 | 预警级别 |
|----------|----------|----------|
| 分包队伍未按时进场 | 实际开始日期 > 计划开始日期 | 红色预警 |
| 分包队伍即将到期 | 计划结束日期 - 当前日期 <= 预警天数（默认7天） | 黄色预警 |
| 分包队伍超期未完工 | 当前日期 > 计划结束日期 且 实际结束日期为空 | 红色预警 |

#### 5.1.2 预警配置

在现有 `warning_setting` 表中增加资源类型：
- 资源类型 7：分包队伍（subcontract）
- 资源类型 8：劳动力（labor）

### 5.2 劳动力需求预警

#### 5.2.1 预警规则

| 预警类型 | 触发条件 | 预警级别 |
|----------|----------|----------|
| 人力不足 | 实际人数 < 计划人数 × 0.8 | 红色预警 |
| 人力即将不足 | 实际人数 < 计划人数 | 黄色预警 |
| 劳动力未按时到位 | 实际开始日期 > 计划开始日期 | 红色预警 |
