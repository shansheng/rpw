## 1. 产品目标

### 1.1 产品愿景

建立一套**智能化、可视化、全流程覆盖**的项目资源计划预警管理系统，实现从资源计划填报、审批流转、智能预警到统计分析的全链路数字化管理，替代传统Excel填报模式。

### 1.2 核心价值

| 价值点 | 描述 |
|--------|------|
| **效率提升** | 在线填报替代Excel，减少人工汇总工作量 |
| **风险可控** | 智能预警提前发现问题，避免资源延误 |
| **流程规范** | 审批流程标准化，责任明确可追溯 |
| **数据洞察** | 多维度统计分析，辅助决策 |

### 1.3 成功指标

- 预警准确率达到95%以上
- 计划变更审批周期缩短50%
- 报表汇总时间从3天缩短到实时

所有8种资源计划类型使用统一的**计划模型**，通过`plan_type`字段区分：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 主键 |

| plan_type | String | 计划类型（material/equipment/hardware/circulation/office/safety/subcontract/labor） |
| status | String | 状态（draft/submitted/approved/in-progress/completed） |

## 6. 预警机制设计

### 6.1 预警规则配置

```
预警设置表 (warning_setting)
├── company_id: 公司ID
├── advance_days: 预警提前天数（如：10 表示提前10天预警）
├── warning_types: 启用的资源类型（JSON数组，1-8）
├── notify_enabled: 是否启用通知（1-启用 0-禁用）
├── notify_channels: 通知渠道（JSON数组：wecom/workwechat）
└── notify_timing: 推送时机（generation-生成预警后立即推送）
```

**通知渠道**：
- **主渠道**：企业微信消息推送（通过企业微信API发送）
- **备用渠道**：系统站内消息

**推送时机**：
- 预警生成后立即推送
- 每日定时扫描（默认每天早上8点）发现新预警时推送

### 6.2 预警触发逻辑

```
┌──────────────────────────────────────────────────────────────┐
│                    每日定时任务 (00:00)                       │
└─────────────────────────────┬────────────────────────────────┘
                              ▼
┌──────────────────────────────────────────────────────────────┐
│  1. 遍历所有预警配置                                           │
│  2. 计算: 预警日期 = 计划日期 - 预警天数                        │
│  3. 判断: 预警日期 == 今日                                      │
│  4. 检查: 该记录是否已处理                                      │
│  5. 生成: 预警记录 (warning_record)                            │
│  6. 推送: 消息通知相关人员                                      │
└──────────────────────────────────────────────────────────────┘
```

### 6.3 预警状态

| 状态 | 说明 | 颜色标识 |
|------|------|----------|
| pending | 待处理 | 红色/黄色高亮 |
| processing | 处理中 | 蓝色 |
| resolved | 已解决 | 灰色 |
| ignored | 已忽略 | 灰色 |

## 8. 附录

### 8.1 WBS Excel导入格式

**文件格式**：Excel（.xlsx / .xls）

**必需列**（按顺序）：

| 列号 | 列名 | 说明 | 示例 |
|------|------|------|------|
| A | 项目名称 | 所属项目名称 | "某大桥项目" |
| B | 类型 | 分部/子单位/单位工程 | "分部工程" |
| C | WBS编号 | 唯一编码 | "WBS-001" |
| D | WBS名称 | 工作包名称 | "基础施工" |
| E | 内容描述 | 工作内容详细说明 | "桩基开挖及混凝土浇筑" |
| F | 形象进度描述 | 可量化的进度指标 | "桩基完成50%" |
| G | 单位 | 计量单位 | "m³" |
| H | 可计数量 | 计划工程量 | "1200" |

**导入逻辑**：
- 系统按"项目名称"自动匹配项目
- WBS编号重复则更新，不重复则新增
- 支持多级WBS（通过parent_id建立层级关系）

### 8.2 术语表

| 术语 | 说明 |
|------|------|
| WBS | Work Breakdown Structure，工作分解结构 |
| 黄色预警 | 距离计划日期还有N天时触发 |
| 红色预警 | 到达计划日期仍未完成时触发 |
| 工序间天数 | 完成当前工序到下一工序开始所需的标准天数 |
