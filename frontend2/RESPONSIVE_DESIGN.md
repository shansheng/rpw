# 响应式设计方案

## 1. 断点定义 (Breakpoints)

基于 Ant Design Vue 的标准断点，并结合项目实际需求：

| 名称 | 断点值 | 设备类型 | Tailwind 前缀 |
| :--- | :--- | :--- | :--- |
| **xs** | < 576px | 手机竖屏 | `sm:` |
| **sm** | ≥ 576px | 手机横屏/小平板 | `sm:` |
| **md** | ≥ 768px | 平板竖屏 | `md:` |
| **lg** | ≥ 992px | 平板横屏/小型桌面 | `lg:` |
| **xl** | ≥ 1200px | 桌面显示器 | `xl:` |
| **xxl** | ≥ 1600px | 大型桌面/宽屏 | `2xl:` |

## 2. 核心响应式策略

### 2.1 布局结构 (Layout)
- **侧边栏 (Sider)**: 
  - 桌面端 (`lg` 及以上): 固定宽度 240px，可折叠。
  - 平板端 (`md` - `lg`): 固定宽度 180px，可折叠。
  - 移动端 (`< md`): 完全隐藏，通过汉堡菜单触发 Drawer 组件从左侧滑出。
- **内容区域 (Content)**: 
  - 始终自适应剩余宽度。
  - 外边距随屏幕尺寸缩小而减小（例如：桌面端 `m-6`，移动端 `m-2`）。
- **顶部导航 (Header)**: 
  - 桌面端：显示完整用户信息、通知、面包屑。
  - 移动端：隐藏面包屑，用户信息简化为头像，增加侧边栏触发按钮。

### 2.2 栅格系统 (Grid System)
- 统一使用 Ant Design Vue 的 `<a-row>` 和 `<a-col>` 组件。
- **卡片/统计区域**: 
  - 桌面端: 4-6 列 (`:span="6"`)
  - 平板端: 2-3 列 (`:span="8"` 或 `:span="12"`)
  - 移动端: 1 列 (`:span="24"`)
- **表单布局**: 
  - 桌面端: 2-3 列表单
  - 平板端: 2 列表单
  - 移动端: 1 列表单，标签独占一行。

### 2.3 表格 (Table)
- **桌面端**: 展示所有列，支持水平滚动。
- **平板/移动端**: 
  - 采用**可横向滚动**的表格 (`scroll="{ x: true }"`)。
  - 或采用**卡片列表**形式展示数据，将不重要的列折叠到抽屉（Drawer）中。
  - 操作列固定在最右侧 (`fixed="right"`)。

### 2.4 字体与排版
- **基准字号**: 桌面端 14px，移动端可微调至 13px 以在有限空间内显示更多内容。
- **标题字号**: 使用 `clamp()` 或 Tailwind 的 `text-sm`, `text-base`, `text-lg`, `text-xl` 等类名进行响应式调整，避免在小屏幕上标题过大。

## 3. 移动端适配方案

### 3.1 触摸优化
- **按钮与链接**: 确保点击区域至少为 44x44px（Ant Design 组件通常已满足）。
- **表格行**: 增加行高，便于手指点击选择。
- **下拉选择**: 在移动端优先使用底部弹出的 `<a-picker>` 或 `<a-action-sheet>` 替代传统的 `<a-select>`，以提升体验。

### 3.2 表单交互
- **日期选择**: 在移动端使用原生的日期选择器或优化后的移动端组件。
- **输入框**: 自动唤起合适的软键盘（如数字键盘、邮箱键盘）。

### 3.3 导航适配
- **面包屑**: 在移动端（`< md`）自动隐藏，通过页面标题替代。
- **标签页 (Tabs)**: 当标签页过多时，允许横向滑动。

## 4. 实现工具与库

- **Tailwind CSS**: 利用其强大的响应式前缀 (`sm:`, `md:`, `lg:`, `xl:`) 快速编写样式。
- **Ant Design Vue**: 组件本身已具备良好的响应式能力（如 `<a-layout>`, `<a-grid>`），优先使用其 API。
- **VueUse**: 使用 `useBreakpoints`, `useMediaQuery` 等 Composable 在 JS 逻辑中响应式地改变行为（例如：根据屏幕宽度决定表格是卡片式还是列表式）。

## 5. 检查清单 (Checklist)

- [ ] 所有页面在不同尺寸浏览器下无横向溢出。
- [ ] 侧边栏在移动端能正常折叠/展开。
- [ ] 表格在移动端可读且可交互。
- [ ] 表单在移动端输入方便。
- [ ] 图片和图表能自适应容器大小。
